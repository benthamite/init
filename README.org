* init.el

Convenience functions to manage Emacs configuration profiles.

** Features

- Create, delete, list, and deploy configuration profiles.
- Tangle configuration files (`config.org`, `user-config.org`) into Elisp files (`init.el`, `early-init.el`, `late-init.el`).
- Handle user-specific configurations and overrides.
- Update configuration from a git repository.
- Integrate with `elpaca` for package management.

** Usage

The main entry point is the `init-menu` transient command, which provides access to common profile management tasks.

*** Profile Management

- ~init-create-profile~: Create a new profile.
- ~init-delete-profile~: Delete an existing profile.
- ~init-list-profiles~: List available profiles.
- ~init-deploy-profile~: Deploy a profile, optionally building it first, then
  smoke-test the generated profile in a fresh background batch Emacs process.
- ~init-build-profile~: Build (tangle) the Elisp files for a profile.

*** Configuration Update

- ~init-update-config~: Pull changes from the configuration repository and offer to deploy a new profile.

*** Package Update

- ~init-update-and-reload~: Update the =init= package itself using =elpaca-extras-update-and-reload=.

** Configuration

- =init-profiles-directory=: Directory where profiles are stored (default: =~/.config/emacs-profiles/=).
- =init-user-config-file=: Path to the user-specific Org configuration file (set via environment variable =USER_CONFIG_FILE=).
- =init-smoke-test-after-deploy=: whether deployment should start a background
  smoke test that verifies the generated profile loads successfully in batch
  Emacs. The smoke-test process suppresses =server-start= so it cannot take over
  or delete the live Emacs server socket.
- =init-smoke-test-emacs-command=: Emacs executable used for deployment smoke
  tests.
- =init-smoke-test-timeout=: seconds before a background smoke test is
  cancelled. The default is long enough for fresh Elpaca profile builds, which
  may need several minutes to clone and compile packages. Reloading the package
  updates an uncustomized live session from the old 120-second default.
- =init-smoke-test-start-codex-on-failure=: whether failed profile smoke tests
  should open a Codex repair session.
- =init-smoke-test-codex-project-directory=: project directory used for Codex
  profile smoke-test repair sessions, defaulting to the dotfiles Emacs
  directory when available.
- =*init-profile-smoke-test*=: public smoke-test result buffer.  Successful
  runs leave only a concise status line; failed runs show only a summary and
  point to the raw output buffer.
- =*init-profile-smoke-test-output*=: raw smoke-test output, kept after a
  failed or timed-out run.
- Hooks: =init-post-init-hook=, =init-post-build-hook=, =init-post-deploy-hook=.
